agent‑bench

An enterprise agent platform earns the right to standardize on a pattern by measuring it, not by taste.

One RAG agent, written once as a declarative spec, run on two frameworks — LangGraph and CrewAI — behind a thin porcelain interface. The platform measures both, and the measurement is what justifies the blended defaults.

A single agent. Two adapters. One eval spine. The model is a swappable seam behind a one-method interface — built so the measuring platform, not any one model, is the product.
▶ Provenance — read this first

Every number on this page is a demonstration number from a deterministic FakeLLM backend — offline, no real model call. This is NOT a real-model benchmark. No GPT, Gemini, or live Claude model was benchmarked to produce these figures. They exist to demonstrate that the measuring workflow runs end to end and is data-driven.

The model is a swappable seam. The same one-method interface accepts the offline FakeLLM, headless Claude Code personas (labeled claude‑as‑<persona>, never passed off as another vendor), or the real Anthropic SDK. The point is model-pluggability: I build the measuring platform; the model is a detail behind a one-method seam.

01   Write once, run on two frameworks

The agent is one declarative spec. Moving it onto the other framework is a one-line change.

  # specs/langgraph-baseline.yaml  →  specs/crewai-baseline.yaml
  # same corpus, same model, same termination policy — only the framework moves

    name: langgraph-baseline
    name: crewai-baseline
-   framework: langgraph
+   framework: crewai
    corpus: corpus
    model: claude-3-5-sonnet-latest
    tools: []
    termination:
      max_iterations: 6
      timeout_s: 30.0
      max_retries: 2
      success_gate: cites_corpus     # answer must cite ≥1 corpus doc

The load-bearing diff is the framework: line. (The name: follows the file, and per-file comments differ — but corpus, model, tools, and the entire termination contract are byte-identical.) Everything an application author declares stays put; only the execution substrate swaps.

success_gate: cites_corpus
— the same declarative contract is handed to both frameworks. The eval then watches how each one honors it.
02   The measurement — both frameworks, same contract

Rendered live from the embedded results.json — data-driven, not hand-typed.

summary section of evalkit/results.json · rates shown as % · latency in seconds

The real signal. On the same two questions under the same cites_corpus gate, crewai gated 2/2 — it always produced a citation and terminated in one iteration — while langgraph gated 1/2: question 2 ran out to max_iter = 6, burned 600 input tokens, and never cited. Same declarative contract, different framework behavior. That is exactly the kind of divergence a platform should standardize on by measuring, not by taste.

Honest caveat. This particular gap is partly a FakeLLM artifact — the deterministic stub drives langgraph's loop to max-iter on q2 in a way a real model might not. The workflow is real; this specific number is not a verdict on either framework.
03   How measurement justifies the defaults

The eval is not a report you read and forget. Its output is wired back to set the porcelain defaults.

The measured gate_rate, iteration counts, and token cost feed the porcelain TerminationPolicy (the max_iterations ceiling, the success_gate) and the retrieval‑k default. A default like max_iterations = 6 stops being a guess and becomes a number with eval provenance attached. Framed honestly: with synthetic n=2 numbers this is a demonstration of the workflow — the wiring is real; the evidence base is intentionally tiny.

measure → annotate the default with where the number came from → ship the default → re-measure. The loop, not any single value, is the deliverable.
04   Architecture

Four layers. The eval spine is the one that earns its keep.

  • porcelainthe thin canonical interface every app author touches — the declarative AgentSpec, the LLM seam (complete()), the retriever, the TerminationPolicy. porcelain/llm.py · retrieval.py · runner.py · types.py
  • adaptersthe per-framework plumbing that honors the spec. adapters/langgraph/ and adapters/crewai/ behind one adapters/base.py.
  • evalkitthe measurement spine — runs both frameworks over the corpus, applies the gate, scores with an LLM-as-judge, emits results.json. evalkit/run.py · judge.py
  • reportrenders the results to results.md and annotates the porcelain defaults with their eval provenance. report/build.py

05   The model-pluggability seam

One method — complete(*, system, messages, model) — three interchangeable clients.

FakeLLM

Offline, deterministic, free. Injected in tests and for these demonstration numbers. No network, no key — the page's data came from here.

claude‑as‑<persona>

Headless Claude Code (claude -p) driven by a persona system prompt. Labeled claude-as-<persona> everywhere — honestly a Claude model wearing a persona, never disguised as another vendor's model.

Anthropic SDK

Production RealAnthropicClient — the real anthropic SDK calling a claude-* model. Lazily imported, so the offline path never needs it installed.

Why this is the point. Swapping the model is a one-line client choice behind a one-method seam. The platform's value is the measuring — the spec, the adapters, the gate, the judge, the provenance. The model is a detail you slot in. That is the posture an enterprise platform team wants: own the measuring surface, treat the model as pluggable.

06   What this maps to in the JD
  • two adapters + one eval→evaluate agent frameworks on the merits, with evidence rather than preference
  • the porcelain interface→design canonical, opinionated APIs that hide framework churn from app teams
  • write-once AgentSpec + one-line swap→developer experience — make the right thing the easy thing
  • the honesty / provenance discipline→trust & safety — synthetic numbers labeled synthetic, personas labeled as Claude, defaults that cite where they came from
  • the model seam→model-pluggability — own the platform, treat the model as a swappable detail
07   Honest limitations
  • Two frameworks only — LangGraph and CrewAI. Not a survey of the field.
  • Tiny fictional corpus — a handful of synthetic ops/runbook docs, not a real knowledge base.
  • Single task shape — RAG-with-citation-gate; one agent pattern, not many.
  • Synthetic numbers — deterministic FakeLLM, not a real-model benchmark.
  • n = 2 questions, 4 runs — far below anything you'd draw a statistical conclusion from. The framework gap is illustrative and partly a FakeLLM artifact.

None of this undercuts the thesis. The deliverable is the measuring platform and the discipline around it — swap in real models and a real corpus, and the same spine produces numbers you could actually standardize on.
